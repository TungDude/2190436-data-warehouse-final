## Big Data Processing Comparison

| Topic | Hadoop (2010's) | Spark + Data Lake (2015-2020) | Serverless (Cloud) / Lakehouse (2020+) |
|---|---|---|---|
| Storage | HDFS | S3 / Data Lake | S3 (files); Iceberg / Delta (metadata; AWS Glue Data Catalog) |
| Compute | MapReduce | Spark (only compute engine!) | Presto/Trino = "read & analyze"; Spark = "build & process"; Flink = "real-time processing" |
| SQL | Hive | Hive / Spark SQL | Athena / BigQuery / Snowflake |

Serverless query flow:

- User (SQL)
  - Athena (service layer)
    - Presto/Trino (execution engine)
      - S3 (storage)
        - Glue Data Catalog (metadata)

## HIVE

- Data Warehouse (SQL Layer) on top of Hadoop
- Easy access to data via SQL
- Enabling data warehousing tasks
  - Extract/Transform/Load (ETL), Reporting, and Data Analysis.
- Hive abstracts the complexity of Hadoop MapReduce -> Tez
  - Hive -> MapReduce -> HDFS
  - Hive -> Tez / Spark -> HDFS/S3
- Hive 3.0 or later fully support CRUD
  - Create, Read, Update, Delete
- Managing Data
  - Managed Table -- Full Analysis
  - External Table -- Ad-hoc Analysis
- Support Both "pre-defined" and "on the fly" Schema

## HIVE vs MapReduce

Example query:

```sql
SELECT s.word, s.freq, k.freq FROM shakespeare s
  JOIN bible k ON (s.word = k.word) WHERE s.freq >= 1 AND k.freq >= 1
  ORDER BY s.freq DESC LIMIT 10;
```

Hive parses the SQL into an Abstract Syntax Tree (AST):

```
(TOK_QUERY (TOK_FROM (TOK_JOIN (TOK_TABREF shakespeare s) (TOK_TABREF bible k) (= (. (TOK_TABLE_OR_COL s) word) (. (TOK_TABLE_OR_COL k) word)))) (TOK_INSERT (TOK_DESTINATION (TOK_DIR TOK_TMP_FILE)) (TOK_SELECT (TOK_SELEXPR (. (TOK_TABLE_OR_COL s) word)) (TOK_SELEXPR (. (TOK_TABLE_OR_COL s) freq)) (TOK_SELEXPR (. (TOK_TABLE_OR_COL k) freq))) (TOK_WHERE (AND (>= (. (TOK_TABLE_OR_COL s) freq) 1) (>= (. (TOK_TABLE_OR_COL k) freq) 1))) (TOK_ORDERBY (TOK_TABSORTCOLNAMEDESC (. (TOK_TABLE_OR_COL s) freq))) (TOK_LIMIT 10)))
```

The AST is then compiled into one or more MapReduce jobs.

## Tez

- Execution Engine
  - Replace MapReduce Engine
- Provides a general-purpose, highly customizable framework
  - Flexible Input-Processor-Output runtime model
  - Creates simplifies data-processing tasks across both
    - small scale (low-latency)
    - large-scale (high throughput)
- Execute complex directed acyclic graphs (DAG) of general data processing
  - Greatly extends the possible ways of which individual tasks can be linked together
  - In fact, any arbitrary DAG can be executed directly in Tez.
- Lazy Operation (similar to Spark)
  - Do not take action immediately
  - Analyze & optimize the whole process
  - Take action whenever it needs!

## HIVE Data Model

- Managed Table (schema -> data)
  - Hive manages the data
  - Create table + Metadata, then then Load data to Table
  - Load data command will move data from HDFS to Hive
  - Drop table command will remove both metadata and data
- External Table (data -> schema)
  - Hive is not responsible for managing the data
  - Only describes the metadata/schema on external data
  - Put data to HDFS, and then create table on top of data
  - Can connect to other databases
    - i.e. HBASE, Druid, ElasticSearch, etc.
  - Drop table command will remove only metadata

## 1) Managed/Internal Table

Drop table, also delete data.

- Store data inside the database engine
- Performance depends on database performance
- Easy to use database utility (cache, ...) to improve performance

Both TABLE (metadata) and DATA live inside the database engine.

*— Slide 11 —*

---

## 2) External Table

Drop table, delete only table NOT data.

- Store data in an external system
  - File System
  - Other databases
- Performance depends on external system (source system)
- Flexible
  - Don't need to load data to table
  - Don't need to create table structure (ad-hoc)
- There are two types of external tables
  - Permanent External Table
  - Ad-hoc External Table (define & delete once it is done).

## Format

- STORED AS TEXTFILE
  - Stored as plain text files.
  - Use the DELIMITED clause to read delimited files.
  - Enable escaping for the delimiter characters (such as ESCAPED BY '\')
  - Custom NULL format can also be specified using the 'NULL DEFINED AS' clause (default is '\N').
- STORED AS SEQUENCEFILE
  - Stored as compressed Sequence File.
- STORED AS ORC (Default in Hive3)
  - Stored as ORC file format (Optimized Row and Columnar format = compressed + RCFILE)
  - Supports ACID Transactions & Cost-based Optimizer (CBO).
  - Stores data block statistic in header, footer
- STORED AS PARQUET
  - Stored as Parquet format for the Parquet columnar storage format
- STORED AS AVRO
  - Stored as Avro format (serializable data)
- STORED AS RCFILE
  - Stored as Record Columnar File format.
- STORED BY
  - Stored by a non-native table format.
  - To create or link to a non-native table, for example a table backed by HBase or Druid or Accumulo.
  - https://cwiki.apache.org/confluence/display/Hive/StorageHandlers

## Parquet

- Columnar storage
- Embed data type
- Compression
- Dev by Google Dremel, Cloudera, Twitter
- Widely used in Cloudera, Impala, Spark
- Compress = smaller data
